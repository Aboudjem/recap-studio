# @recap-studio/html-renderer

Render a typed `RecapPageContent` into **one self-contained, dark-mode, mobile-first HTML file**. No framework, no build step, no runtime JavaScript, no external requests â€” the output **opens with a double-click and works offline**.

This is the reusable visual template behind [Recap Studio](https://github.com/Aboudjem/recap-studio). Any tool that can produce a `RecapPageContent` (or matching JSON) can call it and get a shareable page. See [`TEMPLATE.md`](./TEMPLATE.md) for the reuse + white-label guide.

## Why

Next.js `output: export` writes absolute `/_next/...` asset paths, so a generated page only works behind a web server â€” double-clicking the file shows a blank, unstyled page. This package solves that: it emits a single HTML string with **all CSS inlined** and **zero `<script>`**, so the file is portable, e-mailable, and offline-safe.

## Usage

```ts
import { renderToHtml, renderFromJson } from "@recap-studio/html-renderer";
import content from "./my-recap.json" assert { type: "json" };

// From validated content:
const html = renderToHtml(content, { theme: "dark", animation: "low" });

// From untrusted JSON (parses + validates against the schema, throws on bad input):
const html2 = renderFromJson(JSON.parse(raw));

await Bun.write("recap.html", html); // or fs.writeFileSync
```

## API

| Export | Signature | Notes |
|---|---|---|
| `renderToHtml(content, opts?)` | `(RecapPageContent, RenderOptions) => string` | Pure, synchronous. Returns a full `<!doctype html>` document. |
| `renderFromJson(json, opts?)` | `(unknown, RenderOptions) => string` | Parses + validates first; throws `Invalid RecapPageContent` on mismatch. |
| `getBaseStyles(opts?)` | `(CssOptions) => string` | The stylesheet string, for tools that embed CSS into their own shell. |
| `readingMinutes(content)` | `(RecapPageContent) => number` | Whole-page reading time at 220 wpm. |

### `RenderOptions`

| Option | Type | Default | Meaning |
|---|---|---|---|
| `theme` | `"dark" \| "light" \| "auto"` | `"dark"` | House style is dark. `auto` follows `prefers-color-scheme`. |
| `animation` | `"none" \| "low" \| "medium" \| "high"` | `"low"` | Reveal-on-load intensity. Always reduced-motion safe. |
| `lang` | `string` | `"en"` | `<html lang>`. |
| `attribution` | `string` | `"Generated by Recap Studio"` | Footer line. |

## Guarantees (enforced by tests)

- Output starts with `<!doctype html>` and contains a single inlined `<style>`.
- **No** `<script>` tags, **no** `/_next/` paths, **no** external `.css`/`.js`/image `src`. The only outbound URLs are source-citation links (`target="_blank" rel="noopener"`).
- Dark by default; `prefers-reduced-motion` disables all motion.
- Diagrams: inline `svg` is sanitized (scripts/handlers/external refs stripped) and embedded; Mermaid falls back to an honest static figure (no 3 MB JS).

## Diagrams

For a truly self-contained file, supply diagrams as **hand-authored inline SVG** (`Diagram.kind = "svg"`). Mermaid (`kind = "mermaid"`) renders as a captioned figure with the source in a `<details>` â€” see [`TEMPLATE.md`](./TEMPLATE.md#diagrams).
